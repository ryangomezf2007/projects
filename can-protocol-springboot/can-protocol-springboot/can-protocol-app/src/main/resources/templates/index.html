1 Introduction + Need of CAN

What is CAN Protocol?
And Why Was It Invented?

CAN — The Car's Internal Internet

CAN (Controller Area Network) is a communication system that allows different computers (ECUs) inside a car to talk to each other without a central host. Developed by Bosch in 1986 and standardized as ISO 11898, it became mandatory in all cars sold in the USA from 2008.

💡
A modern car has 70+ ECUs. Without CAN, they'd each need separate wire pairs — thousands of wires, hundreds of kilograms of copper.
🏫
Analogy
CAN is like a common notice board in a school. Any student (ECU) can post a notice (message). Every other student can read it. No one needs a personal messenger.

Problems Before CAN

❌ Before CAN
Separate wire for every sensor/actuator pair
Engine → Dashboard needed its own wire
Brake → ABS needed its own wire
Result: 2km+ of wiring in a car
Heavy, expensive, failure-prone
✅ With CAN
All ECUs share just 2 wires
Any ECU can send/receive any message
Plug-in new ECU = instant connectivity
Under 100m total wiring length
Lighter, cheaper, more reliable

2 CAN Bus + How Communication Works

The CAN Bus
Two Wires. Infinite Conversations.

CAN-H (High)

In a dominant bit: ~3.5V
In a recessive bit: ~2.5V

Carries the positive component of the differential signal
Differential Voltage
Vdiff = CANH − CANL → noise immune!

CAN-L (Low)

In a dominant bit: ~1.5V
In a recessive bit: ~2.5V

Carries the negative component — together they reject noise

How Communication Happens — Step by Step

1
Engine ECU sends a message
Engine ECU puts a message on the bus: ID=0x100 | DATA: Speed=80km/h, RPM=3000
2
All ECUs receive the message
Every ECU on the bus receives the frame. It's like a broadcast — no one needs to be "called".
3
Each ECU decides: relevant or not?
Dashboard ECU: ID=0x100 means speed → show 80 km/h
Gear ECU: ID=0x100 → adjust gear ratio
Airbag ECU: not relevant → ignore
4
Bus becomes free again
After the message frame ends, any ECU may now transmit. The cycle repeats thousands of times per second.

4 Arbitration + Error Handling

Non-Destructive Arbitration
Who Gets the Bus?

When two ECUs transmit simultaneously, CAN resolves the conflict through bitwise arbitration — without corrupting either message. The lower ID always wins.

🛡️ Airbag ECU
ID = 0x050 (binary: 000 0101 0000)
Waiting…
⚡ VS ⚡
📻 Radio ECU
ID = 0x3FF (binary: 111 1111 1111)
Waiting…
CAN Bus — Wired-AND Logic
🔗 Wired-AND: if ANY node sends dominant (0), the bus reads 0
🏆 Dominant (0) = wins arbitration | Recessive (1) = loses

5 Layers of Error Detection

🔍

Bit Monitoring

Transmitter reads back its own sent bits. If sent 1 but reads 0 → error flag issued immediately.

TX: 1 | RX: 0 → ERROR!
📦

Bit Stuffing

After 5 identical bits, a complementary bit is inserted. Missing stuff bit = error. Keeps signal transitions regular.

11111000000 ← stuffed
🔐

CRC Check

15-bit polynomial checksum. Receiver recalculates and compares. Detects up to 5 random bit errors anywhere in the frame.

Tx CRC ≠ Rx CRC → ERROR
🏗️

Frame Check

EOF, ACK delimiter, CRC delimiter must have specific bit patterns. Any violation triggers an error frame.

EOF ≠ 7 recessive → ERROR
✅

ACK Error

Transmitter expects a dominant ACK bit from receivers. If it sees recessive → nobody confirmed → retransmit!

ACK slot = 1 → RETRANSMIT

Error Confinement — Faulty Node Isolation

Error Active
Normal operation. TEC/REC < 96. Node can participate fully.
→
Error Passive
TEC or REC > 127. Node can still receive but can't actively disrupt bus.
→
Bus Off
TEC > 255. Node disconnects from bus automatically. Others unaffected!
🚨
Analogy
CAN arbitration is like a road with an ambulance. When a critical message (airbag) and a low-priority message (radio) try to go simultaneously, the road system (bus) automatically gives right-of-way to the ambulance (lower ID). No crash. No delay.

Why CAN Wins — 5 Key Advantages

✅

Less Wiring

All ECUs share 2 wires. A car goes from 2km+ of wiring down to under 100m. Lighter = better fuel economy.

✅

Low Cost

Fewer wires, cheaper connectors, simpler PCB routing. Mass production drives chip costs under $1 USD per node.

✅

Fast Communication

Up to 1 Mbps (CAN FD: 8 Mbps). A car processes 2,000+ messages per second reliably across all ECUs.

✅

Highly Reliable

Differential signaling rejects electrical noise. Operates in harsh environments: engine bay heat, vibration, EMI.

✅

Built-in Error Detection

5 independent error checks per frame. Faulty nodes self-isolate. The bus keeps running even when one node fails.
